# Лабораторная работа №4

## Диагностика компьютера с помощью программ-диагностик

**Цель работы:** освоить методику комплексной диагностики ПК с использованием программы PC Wizard, научиться анализировать полученные данные о конфигурации и производительности системы.

---

### Общие сведения о диагностическом ПО

Программы для диагностики компьютера позволяют:

- **Проверить конфигурацию компьютера:**
    - объём памяти и характер её использования;
    - типы установленных накопителей;
    - другие аппаратные и системные параметры.

- **Проверить работоспособность устройств** (особенно жёстких дисков и SSD).

- **Выявить «зарождающиеся» дефекты** дисков (износ магнитного слоя или ячеек флеш-памяти) и предотвратить потерю данных.

- **Определить несложные неисправности** оборудования или ошибки в конфигурации системных файлов.

### Объекты диагностики

Диагностика выполняется как для системы в целом, так и для отдельных модулей:

| Категория | Компоненты |
|-----------|------------|
| Системные компоненты | Материнская плата, процессор, чипсет |
| Память | Оперативная память (все типы) |
| Видеоподсистема | Интегрированная и дискретная графика |
| Накопители | HDD, SSD, оптические приводы |
| Устройства ввода | Клавиатура, мышь, тачпад |
| Интерфейсы и порты | COM, LPT, USB, SCSI, Thunderbolt |
| Периферия | Принтеры, сканеры, модемы, сетевые адаптеры |

---

### Программа PC Wizard

**Описание**  
PC Wizard — это информационно-диагностическая утилита, предоставляющая подробные сведения о всех компонентах компьютера.

**Возможности программы:**

- Сбор информации об оборудовании:
    - оперативная память (тайминги, частота, объём);
    - материнская плата (модель, чипсет, версия BIOS);
    - накопители (HDD, SSD, оптические приводы);
    - видеоподсистема (GPU, объём видеопамяти, драйвер);
    - сетевые устройства (LAN, Wi‑Fi, Bluetooth);
    - принтеры и другая периферия.

- Сбор информации об операционной системе:
    - версия и сборка ОС;
    - установленные шрифты, библиотеки (DLL);
    - параметры WinSock;
    - активные процессы, модули и службы.

- Тестирование производительности:
    - быстродействие процессора (целочисленные и вещественные операции);
    - скорость чтения/записи жёстких дисков и SSD;
    - производительность CD/DVD/Blu‑Ray привода.

**Системные требования:**  
- Поддержка 32-битных (x86) и 64-битных (x64) операционных систем.

## Ответы на контрольные вопросы

### Вопрос 1: Причины возникновения проблем с совместимостью ПО

| Категория | Конкретные причины |
|:---|:---|
| **Аппаратные** | • Слишком старый процессор (отсутствие нужных инструкций — SSE4, AVX). <br> • Недостаток оперативной памяти (например, 2 ГБ для Windows 10). <br> • Отсутствие драйверов под новую ОС (особенно для редких устройств). <br> • Несоответствие разрядности программы и ОС (32‑битное ПО на 64‑битной ОС обычно работает, но не наоборот). |
| **Программные** | • Устаревшая версия ОС (например, Windows 7 не поддерживает DirectX 12). <br> • Отсутствие нужных библиотек: .NET Framework, Visual C++ Redistributable, Java, DirectX. <br> • Конфликт версий: программа требует старую версию библиотеки, а установлена новая. <br> • Изменение API между версиями ОС (функция удалена или работает иначе). |
| **Системные / настройки** | • Ограничения безопасности: UAC блокирует запись в системные папки или реестр. <br> • Антивирус карантинит файлы программы. <br> • Несоответствие региональных настроек (формат даты, десятичный разделитель). <br> • Проблемы с правами доступа (программа требует администратора). |

### Вопрос 2: Методы уменьшения проблем с совместимостью ПО

| Метод | Как помогает |
|:---|:---|
| **Обновление «железа»** | Добавление ОЗУ, замена HDD на SSD, установка современной видеокарты — решает проблемы с производительностью и поддержкой драйверов. |
| **Обновление ПО** | Установка последних обновлений Windows, драйверов и библиотек — закрывает большинство совместимостей. |
| **Режим совместимости Windows** | Правый клик по .exe → «Свойства» → «Совместимость» → запуск от имени Windows 7/XP. Помогает для старых программ. |
| **Запуск от имени администратора** | Даёт программе права на запись в защищённые области (Program Files, реестр, системные папки). |
| **Виртуальные машины (VirtualBox, VMware)** | Установка внутри виртуальной машины старой ОС (например, Windows XP) для запуска очень старого ПО. |
| **Эмуляторы** | DOSBox — для программ под DOS; Wine — для запуска Windows-приложений на Linux. |
| **Контейнеризация / песочница** | Windows Sandbox (встроенная) или Sandboxie — изолированный запуск потенциально проблемных программ. |
| **Поиск современного аналога** | Вместо старой программы найти новую, которая делает то же самое, но совместима с текущей ОС. |
| **Двойная загрузка (Dual Boot)** | Установка на компьютер двух ОС — современной и старой. При загрузке выбирается нужная. |
| **Портативные версии (Portable)** | Программы, которые не требуют установки и не пишут в реестр. Меньше конфликтов с системой. |
| **Ручная настройка** | Отключение сложных визуальных эффектов Aero/DWM, изменение масштаба DPI, ограничение выделяемой видеопамяти и т.д. |
