# Практическая работа №9

## Изучение ГОСТ 34.602-89: Техническое задание на автоматизированную систему

**Цель работы:** ознакомиться с требованиями государственного стандарта ГОСТ 34.602-89 к составу, содержанию и оформлению документа «Техническое задание на создание автоматизированной системы (АС)», а также освоить практические аспекты разработки ТЗ.

---

## 1. Общие сведения о стандарте

ГОСТ 34.602-89 является действующим межгосударственным стандартом, который применяется при разработке автоматизированных систем различного назначения:
- системы управления (предприятием, технологическими процессами);
- системы проектирования (САПР);
- научно-исследовательские системы;
- информационно-справочные системы.

Стандарт устанавливает единые требования к документу **«Техническое задание на создание (развитие или модернизацию) системы»** (сокращённо — ТЗ на АС).

**Ключевая особенность:** стандарт рассматривает АС как комплексную систему, включающую не только программное и техническое обеспечение, но также:
- персонал (пользователей, администраторов);
- автоматизируемые бизнес-процессы;
- организационно-распорядительную документацию.

### Назначение ТЗ

| Функция | Описание |
|:--------|:---------|
| **Фиксация требований** | ТЗ закрепляет все требования заказчика к будущей системе. |
| **Основа для разработки** | Все проектные решения принимаются на основе утверждённого ТЗ. |
| **Юридическая база** | ТЗ является приложением к договору и имеет юридическую силу. |
| **Критерий приёмки** | Система считается выполненной, если она соответствует ТЗ. |

---

## 2. Состав разделов технического задания (по ГОСТ 34.602-89)

Согласно стандарту, ТЗ на АС должно содержать следующие разделы (допускается объединение, исключение или добавление подразделов по согласованию с заказчиком):

| № | Наименование раздела | Краткое содержание |
|---|----------------------|---------------------|
| 1 | Общие сведения | Наименование системы, реквизиты сторон, сроки, финансирование. |
| 2 | Назначение и цели создания системы | Для чего нужна система и каких результатов ожидают. |
| 3 | Характеристика объекта автоматизации | Описание существующей ситуации (что автоматизируется). |
| 4 | Требования к системе | **Самый большой раздел** — функциональные и нефункциональные требования. |
| 5 | Состав и содержание работ по созданию системы | План работ: этапы, сроки, исполнители, документы. |
| 6 | Порядок контроля и приёмки системы | Как будут проверять и принимать систему. |
| 7 | Требования к подготовке объекта автоматизации | Что нужно сделать на стороне заказчика перед запуском. |
| 8 | Требования к документированию | Какие документы должны быть разработаны. |
| 9 | Источники разработки | Какие исходные материалы использовались. |

> **Примечание:** стандарт допускает добавление приложений (например, расчёт эффективности, перечень нормативных документов).

### Раздел 3. Характеристика объекта автоматизации

**Что описывается:**
- Организационная структура предприятия;
- Существующие бизнес-процессы (как работают сейчас);
- Объёмы обрабатываемой информации;
- Условия эксплуатации (температура, влажность, режим работы).

**Пример:**
> Объектом автоматизации является складской комплекс ООО «Торг-Сервис» площадью 2000 м². В настоящее время учёт ведётся вручную в журналах и таблицах Excel. Штат кладовщиков — 5 человек, режим работы — 5 дней в неделю с 9:00 до 18:00.

### Раздел 4. Требования к системе (ключевой раздел)

Состоит из трёх подразделов:

| Категория | Пример требования |

| Эргономика | Интерфейс должен быть интуитивно понятным для пользователя с базовыми навыками ПК. |

#### 4.2. Требования к функциям (задачам)

| Функция | Временной регламент | Точность |
|:--------|:--------------------|:---------|
| Ввод накладной | не более 30 секунд | 100% целостность данных |
| Поиск товара | не более 2 секунд | 100% попадание |
| Формирование отчёта | не более 10 секунд | погрешность < 0,1% |

#### 4.3. Требования к видам обеспечения

| Вид обеспечения | Требования |
|:----------------|:-----------|
| **Программное** | ОС: Windows Server 2022; СУБД: PostgreSQL 15; язык программирования: C# / ASP.NET Core. |
| **Техническое** | Сервер: 8 ядер / 32 ГБ RAM / 2×1 ТБ RAID 1; клиенты: ПК с ОС Windows 10/11. |
| **Информационное** | Обмен данными — через REST API; классификаторы — по ОКПО, ОКВЭД. |
| **Организационное** | Назначить администратора системы (из штата заказчика). |
