# Лабораторная работа №2
## ОЦЕНКА КАЧЕСТВА РАБОТЫ НАКОПИТЕЛЯ (HDD/SSD)

**Цель работы:** освоить методику оценки производительности и технического состояния жёсткого диска или твердотельного накопителя с помощью специализированных утилит.

### Теоретическая часть

**CrystalDiskMark** — утилита для измерения скорости последовательного и случайного чтения/записи. Позволяет задавать размер блоков данных, глубину очереди команд (Queue Depth) и количество потоков (Threads). Благодаря этому можно оценить производительность накопителя в разных сценариях: при работе с крупными файлами (SEQ) или с мелкими блоками (RND4K).

**CrystalDiskInfo** — программа для мониторинга состояния накопителя. Считывает данные S.M.A.R.T. (Self-Monitoring, Analysis and Reporting Technology), отображает температуру, количество ошибок, время наработки, число включений и другие параметры. Позволяет своевременно обнаружить признаки деградации диска.

### Порядок выполнения работы

1. Установить программу CrystalDiskMark.
2. Запустить утилиту.

4. Нажать кнопку **All** для запуска всех тестов.

### Контрольные вопросы

**1. Что такое жёсткий диск?**

Жёсткий диск (Hard Disk Drive, HDD) — это устройство для долговременного хранения данных, работающее на принципе магнитной записи. В отличие от оперативной памяти, он энергонезависим — информация сохраняется после выключения питания. Сегодня наряду с HDD широко используются SSD (твердотельные накопители), у которых нет движущихся частей.

**2. Внутреннее устройство жёсткого диска**

Классический HDD (с магнитными пластинами) состоит из следующих частей:

- **Гермоблок** — пылезащищённый герметичный корпус.
- **Магнитные пластины** — диски из алюминия или стекла с магнитным слоем.
- **Шпиндельный двигатель** — раскручивает пластины до 5400, 7200 или 10000 об/мин.
- **Блок магнитных головок** — считывает и записывает данные.
- **Актуатор (позиционер)** — перемещает головки над пластинами.
- **Контроллер** — печатная плата, управляющая работой диска и взаимодействием с компьютером.

У SSD внутреннее устройство иное: используются микросхемы флеш-памяти (NAND) и контроллер, механических частей нет.

**3. Причины возникновения проблем с совместимостью накопителей**

Основные причины проблем совместимости:

- Устаревший интерфейс (например, IDE вместо SATA).
- Ограничения BIOS/UEFI при работе с дисками большого объёма (>2 ТБ на старых системах).
- Неподходящая файловая система (например, диск отформатирован в ext4, а система работает под Windows).
- Отсутствие драйверов для NVMe-накопителей в старых ОС (например, Windows 7).
- Конфликты на одной шине (SATA-порт может быть отключён при использовании M.2).

**4. Методы уменьшения проблем с совместимостью**

Чтобы снизить риск проблем, рекомендуется:

- Обновить BIOS/UEFI материнской платы до последней версии.
- Установить актуальные драйверы чипсета и контроллеров.
- Использовать режим AHCI вместо IDE для SATA-дисков.
- При необходимости применять адаптеры (например, M.2 → SATA или SATA → USB).
- Перед установкой ОС загрузить драйверы для NVMe (для Windows 7 и старше).
- Для старых ПК с BIOS использовать диски объёмом не более 2 ТБ либо ставить специальные загрузчики (Clover, GRUB).
